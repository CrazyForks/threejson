# ThreeJSON Project 商标与品牌政策

[English](./TRADEMARK.md)

**版本 1.0——最后更新：2026-07-11**

## 1. 政策目的

ThreeJSON 是一个开源项目。对于明确采用 MIT License 的源代码及其他材料，任何人均可在遵守该许可证的前提下使用、复制、修改、分发、再许可及商业使用。

本《商标与品牌政策》仅适用于用于识别 ThreeJSON Project 官方项目及其官方产品的名称和品牌资产。

**本政策不会修改、替代或限制 MIT License，或明确适用于特定文件、资源及材料的其他许可证所授予的任何权利。**

## 2. 定义

在本政策中：

- **“ThreeJSON Project”**，是指 ThreeJSON 官方项目，以及通过官方网站或官方仓库公开标明的相关品牌权利人或经授权的品牌管理者。
- **“官方项目渠道”**，是指通过 ThreeJSON 官方网站或官方仓库链接或明确列出的官方网站、仓库、组织、软件包发布账号及其他官方账号。
- **“品牌资产”**，是指主要用于识别 ThreeJSON Project 或其某项官方产品的官方 Logo、文字标识、Logo 组合标识及品牌专用图标。

## 3. 本政策涵盖的官方名称

本政策目前涵盖的官方项目及产品标识包括：

- ThreeJSON
- ThreeBox
- ThreeJSON Core
- ThreeJSON Domain
- ThreeJSON SceneEditor
- ThreeJSON ScenePlayer
- ThreeJSON SceneShower
- ThreeBox Community

其他名称只有在通过官方项目渠道公开使用或被明确指定为官方名称时，才属于本政策的适用范围。

本政策不主张对单独使用的普通词语或通用概念享有排他性权利，包括：

- core；
- domain；
- plugin；
- editor；
- player；
- scene；
- shower；
- community；
- cloud；
- pro；
- plus；
- ultimate。

特别是，本政策不限制“domain”一词的普通使用，也不限制用户通过 ThreeJSON 扩展 API 创建自定义 Domain。

## 4. 源代码与开放许可材料

在遵守相关许可证的前提下，您可以：

- 使用 ThreeJSON 源代码；
- 将其用于商业软件、专有软件或内部应用；
- 对明确采用 MIT License 的材料进行复制、修改、合并、发布、分发、再许可或销售；
- 私有部署修改后的版本；
- 创建 Fork 和衍生作品；
- 基于 ThreeJSON 构建商业产品与服务。

软件仅仅使用了 ThreeJSON 源代码或其他采用开放许可证的项目资源，并不需要另行取得商标许可。

本政策只管理品牌及官方身份，不会对明确采用开源许可证或开放内容许可证的代码及其他材料附加额外限制。

## 5. 普通项目资源

本政策不会自动适用于普通项目资源，包括：

- 纹理及材质贴图；
- 3D 模型；
- 场景资源；
- Demo 图片；
- 示例文件；
- 截图；
- 普通用户界面图标；
- 音频、视频及其他非品牌资源。

此类材料应按照相关文件、目录或仓库中附带的许可证或授权声明使用。

明确采用 MIT License 或其他开放许可证的普通项目资源，可以按照相应许可证使用。它们不会仅仅因为被收录在 ThreeJSON 官方仓库中，就自动成为受到本政策限制的品牌资产。

第三方资源继续适用其各自的许可证、署名要求及其他声明。

## 6. 描述性使用与社区使用

在内容真实、准确且不会造成误导的情况下，您可以无须事先许可使用 ThreeJSON 和 ThreeBox 名称，例如：

- Built with ThreeJSON
- Powered by ThreeJSON
- Uses ThreeJSON
- Compatible with ThreeJSON
- Supports ThreeJSON
- Based on ThreeJSON
- Forked from ThreeJSON

您还可以将相关名称用于：

- 链接至官方网站或官方仓库；
- 介绍、讨论、教学、评测、比较或报道本项目；
- 说明兼容性或互操作性；
- 准确描述第三方项目与官方项目的关系；
- 说明某个 Fork 或衍生作品的上游项目来源。

除非相关关系真实存在，否则上述使用不得暗示第三方项目属于官方项目，或获得 ThreeJSON Project 的认可、赞助、认证、维护、批准或其他官方关联。

## 7. Domain、Plugin 与其他扩展

ThreeJSON 被设计为一个可扩展的平台。

开发者可以创建、发布并商业使用自己的：

- Domain；
- Plugin；
- Component；
- Script；
- Runtime Extension；
- Adapter；
- AI Integration；
- Tool；
- 其他基于 ThreeJSON 公开 API 构建的集成与扩展。

本政策不主张对第三方 Domain、Plugin、集成或其他扩展享有所有权。其权利归属及许可方式，由相关作者、贡献者、雇主、合同、适用许可证及其他协议决定。

在遵守相关软件和资源许可证的前提下，用户可以自由定义、注册、发布并商业使用自定义 ThreeJSON Domain。

建议第三方项目采用能够明确体现其第三方来源的名称，例如：

- Acme Factory Domain for ThreeJSON
- Acme ThreeJSON Adapter
- `@acme/threejson-plugin`
- `acme-threejson-domain`

当第三方来源清晰，且整体展示方式不会使公众误认为其属于官方项目时，第三方可以在项目名或软件包名称中以描述性方式使用 ThreeJSON 或 ThreeBox。

下列名称在可能使公众误认为属于官方项目时，需要事先取得书面许可：

- ThreeJSON Official Factory Plugin
- Official ThreeBox Extension
- ThreeJSON Core
- ThreeJSON SceneEditor

本段限制的是具有误导性的官方身份表达，而不是兼容性说明或普通的描述性使用。

## 8. Fork 与修改后的发行版

相关开源许可证允许 Fork 和修改本项目。

如果托管平台明确将某个仓库标识为 Fork，该仓库可以保留平台生成或允许使用的仓库名称。

当修改后的版本被独立发布、推广或分发时：

- 不得将其描述为 ThreeJSON 或 ThreeBox 官方发行版；
- 应当清楚说明实质性修改；
- 应当使第三方维护者或分发者的身份保持合理清晰。

如果整体展示方式可能使用户误认为修改后的发行版属于官方版本，则不得将 ThreeJSON 或 ThreeBox 作为该发行版的主要品牌。

对于独立维护或经过实质修改的发行版，强烈建议采用不同的主要名称，同时可以准确描述其关系，例如：

- Forked from ThreeJSON
- Based on ThreeJSON
- Powered by ThreeJSON

## 9. 仓库、软件包、域名、组织与服务

ThreeJSON 和 ThreeBox 是通过官方项目渠道维护的官方项目身份标识。

未经事先书面许可，不得以可能使用户合理误认为具有下列身份的方式，使用 ThreeJSON、ThreeBox、官方产品名称或容易与其混淆的名称，创建或运营第三方仓库、软件包、发行版、网站、服务、产品、公司、组织或账号：

- 官方项目或官方发行版；
- 由 ThreeJSON Project 维护或发布；
- 获得 ThreeJSON Project 的认可、赞助、认证、批准或支持；
- 与 ThreeJSON Project 存在其他官方关联。

该原则适用于可能造成误导的下列使用场景：

- GitHub 仓库和 GitHub Organization；
- npm 及其他软件包注册表；
- 域名和网站；
- 应用程序及服务名称；
- 可下载发行版；
- 公司及组织名称；
- 社交媒体及社区账号；
- 云服务及托管服务。

当第三方身份清晰时，本节不禁止 `@acme/threejson-adapter`、“Compatible with ThreeJSON”或“Unofficial community tool for ThreeJSON”等准确的描述性使用。

## 10. 需要事先取得许可的使用方式

下列行为需要事先取得书面许可：

- 将第三方项目、产品或服务描述为 ThreeJSON 或 ThreeBox 官方产品；
- 将官方品牌资产用作第三方产品、公司、组织或服务的主要 Logo 或主要身份标识；
- 以可能暗示并不存在的认可、赞助、认证、合作或官方关联的方式使用官方品牌；
- 注册或运营其名称和展示方式可能与 ThreeJSON 或 ThreeBox 官方身份产生混淆的商标、企业名称、域名、软件包、仓库、组织或账号；
- 将修改后的版本作为官方发行版进行分发；
- 在商品或宣传材料中，以可能使公众误认为具有官方来源或获得官方认可的方式使用官方品牌资产。

## 11. 品牌资产

除非另有明确许可，MIT License 不授予将官方品牌资产用作第三方产品、服务、公司、组织或项目身份标识的权利。

在符合下列条件时，可以将官方品牌资产用于真实的编辑、教育、记录、比较或指代目的：

- 为识别本项目而进行相关使用具有合理必要性；
- 不将相关品牌资产用作第三方自己的 Logo 或主要身份标识；
- 展示方式不会暗示并不存在的官方关系。

软件、网站、示例或渲染场景的截图不会自动被视为品牌资产。截图的使用适用相关资源许可证，并且不得用于虚假暗示官方关联。

## 12. 官方身份与冒充行为

只有通过官方项目渠道链接或明确列出的仓库、软件包、网站、账号、发行版、产品及服务，才应被视为官方内容。

仓库、软件包、域名、组织或账号名称中仅仅出现 ThreeJSON、ThreeBox 或相关词语，并不当然意味着其属于官方项目。

任何人不得利用官方名称或品牌资产冒充 ThreeJSON Project，不得虚假描述软件或服务的来源，不得将用户引导至具有欺骗性的下载内容，也不得在来源、维护、安全性、认可或关联关系方面误导用户。

## 13. 善意解决原则

ThreeJSON Project 制定本政策的目的，是保护用户免受冒充和混淆，而不是阻碍正当的开源使用、互操作性、社区扩展、评论介绍或商业采用。

在可行的情况下，对于明显出于善意的问题，通常会首先要求相关使用者进行合理澄清、补充署名、修改名称或采取其他纠正措施。

对于欺诈、故意冒充、恶意软件、欺骗性分发、安全风险、重复违规或可能造成严重用户混淆的行为，可以立即采取更严格的措施。

## 14. 政策更新

随着项目及其治理方式的发展，本政策可能进行更新。

重大修改将通过官方项目渠道公开发布。除非为了防止欺诈、冒充、安全损害或严重用户混淆而有合理必要立即采取措施，对于原本符合旧版政策的既有使用，将给予合理的调整期限。

ThreeJSON Project 在某一次具体情况下未执行本政策，不代表放弃在其他情况下执行本政策的权利。

## 15. 权利保留

ThreeJSON Project 保留本政策未明确授予的全部商标、品牌及官方身份相关权利。

本政策中的任何内容，均不授予任何人对官方名称或品牌资产的所有权，也不授予将其注册为商标、企业标识、域名、账号、软件包命名空间或其他排他性标识的权利。
